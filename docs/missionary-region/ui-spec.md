## 1. 설계 결정 (Design Decisions)

### 등록/수정 폼: 모달 채택 (슬라이드 패널 대신)

| 기준 | 슬라이드 패널 | 모달 |
|------|:---:|:---:|
| 필드 수 | 많을 때 유리 (10+) | 적을 때 유리 (6개) ✓ |
| 목록 맥락 유지 | O | X |
| 복잡도 | 높음 | 낮음 ✓ |
| 유저 관리 패턴 일관성 | O | X |

**결정**: 연계지 폼은 필드 6개로 단순하고, 목록과 동시 확인 필요성이 낮아 **모달**을 채택한다.
등록과 수정은 동일한 폼 컴포넌트를 재사용한다 (제목만 분기).

## 3. 화면별 상세 설계

### 3-1. 메인 페이지

```
┌─────────────────────────────────────────────────────────────────────┐
│  연계지 관리                              [+ 연계지 등록] (ADMIN)  │
├─────────────────────────────────────────────────────────────────────┤
│  선교                                                               │
│  [Select: 선교를 선택하세요              ▼]                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ← 선교 미선택 시: 빈 상태 A             │
│  ← 연계지 없음 시: 빈 상태 B             │
│  ← 연계지 있음 시: 테이블                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**헤더**
- 좌: 페이지 제목 "연계지 관리" (h1)
- 우: `[+ 연계지 등록]` 버튼
  - ADMIN 전용 노출 (일반 사용자에게 완전 미렌더)
  - 선교가 선택되지 않은 상태에서는 `disabled` 처리 + tooltip: "선교를 먼저 선택해주세요"

**선교 선택 영역**
- 레이블: "선교"
- 컴포넌트: `Select` (기존 디자인 시스템 컴포넌트)
- placeholder: "선교를 선택하세요"
- 옵션: Missionary 목록 (이름 표시)
- 상태: URL 쿼리 파라미터 `missionaryId`로 관리 (새로고침 시 유지)

### 3-3. 연계지 목록 테이블

```
┌────────┬──────────┬────────┬──────────────┬────────────────────────┬──────────────┐
│  이름  │ 방문목적  │ 목사명 │  목사연락처  │          주소          │    액션      │
├────────┼──────────┼────────┼──────────────┼────────────────────────┼──────────────┤
│ OO교회 │ 단기선교  │ 홍길동 │ 010-1234-    │ 서울 강남구 테헤란로   │ [수정][삭제] │
│        │          │        │ 5678         │ 123 / 2층 201호        │   (ADMIN)    │
├────────┼──────────┼────────┼──────────────┼────────────────────────┼──────────────┤
│ △△교회 │ -         │ -      │ -            │ -                      │ [수정][삭제] │
└────────┴──────────┴────────┴──────────────┴────────────────────────┴──────────────┘
```

**컬럼 명세**

| 컬럼 | width | 정렬 | 비고 |
|------|-------|------|------|
| 이름 | 160px | 좌 | 필수값, 항상 존재 |
| 방문목적 | 160px | 좌 | 없으면 `-` 표시 |
| 목사명 | 120px | 좌 | 없으면 `-` 표시 |
| 목사연락처 | 140px | 좌 | 없으면 `-` 표시 |
| 주소 | 280px (flex) | 좌 | 기본주소 + 상세주소 줄바꿈 표시, 모두 없으면 `-` |
| 액션 | 100px | 중앙 | ADMIN 전용 렌더 |

**주소 표시 규칙**
- 기본주소만 있으면: `서울 강남구 테헤란로 123`
- 기본+상세 있으면: `서울 강남구 테헤란로 123` (줄바꿈) `2층 201호` (text-muted-foreground, text-sm)
- 모두 없으면: `-`

**액션 버튼 (ADMIN 전용)**
- `[수정]`: `Button` variant=`outline` size=`sm` 또는 `IconButton`(pencil 아이콘)
- `[삭제]`: `Button` variant=`outline` size=`sm`, destructive 색상 또는 `IconButton`(trash 아이콘)
- 버튼 간격: `gap-2`

**일반 사용자**: 액션 컬럼 자체를 렌더하지 않음

### 3-4. 등록/수정 모달

```
┌─────────────────────────────────────────────┐
│  연계지 등록              (수정 시: 연계지 수정)  [×] │
├─────────────────────────────────────────────┤
│                                             │
│  이름 *                                     │
│  ┌─────────────────────────────────────┐   │
│  │ 연계지(교회) 이름을 입력하세요       │   │
│  └─────────────────────────────────────┘   │
│                                             │
│  방문목적                                   │
│  ┌─────────────────────────────────────┐   │
│  │ 예) 단기선교, 교류 방문             │   │
│  └─────────────────────────────────────┘   │
│                                             │
│  목사명            목사연락처               │
│  ┌──────────────┐  ┌──────────────────┐   │
│  │ 담당 목사 이름│  │ 010-0000-0000    │   │
│  └──────────────┘  └──────────────────┘   │
│                                             │
│  기본주소                                   │
│  ┌─────────────────────────────────────┐   │
│  │ 도로명 주소 또는 지번 주소          │   │
│  └─────────────────────────────────────┘   │
│                                             │
│  상세주소                                   │
│  ┌─────────────────────────────────────┐   │
│  │ 건물명, 층, 호수 등                 │   │
│  └─────────────────────────────────────┘   │
│                                             │
├─────────────────────────────────────────────┤
│                     [취소]   [저장]         │
└─────────────────────────────────────────────┘
```

**모달 명세**

| 속성 | 값 |
|------|-----|
| 너비 | 480px |
| 오버레이 | 반투명 dimmed overlay |
| 관리 방식 | overlay-kit |
| 닫기 | [×] 버튼 또는 오버레이 클릭 (단, dirty 상태면 이탈 확인 모달) |

**폼 필드 명세**

| 필드 | 컴포넌트 | 필수 | 검증 | placeholder |
|------|---------|:----:|------|------------|
| 이름 | `InputField` | Y | `min(1, '이름을 입력해주세요')` | 연계지(교회) 이름을 입력하세요 |
| 방문목적 | `InputField` | N | - | 예) 단기선교, 교류 방문 |
| 목사명 | `InputField` | N | - | 담당 목사 이름 |
| 목사연락처 | `InputField` | N | 전화번호 형식 (선택적) | 010-0000-0000 |
| 기본주소 | `InputField` | N | - | 도로명 주소 또는 지번 주소 |
| 상세주소 | `InputField` | N | - | 건물명, 층, 호수 등 |

**레이아웃 규칙**
- 목사명 + 목사연락처: 2열 그리드 (`grid grid-cols-2 gap-4`) — 의미적으로 연관된 필드 묶음
- 나머지 필드: 1열 (full width)

**버튼**
- `[취소]`: `Button` variant=`outline` — 닫기 (dirty 상태면 이탈 확인)
- `[저장]`: `Button` variant=`default` — form submit, loading 상태 지원

**수정 모달 초기값**: 선택된 연계지의 현재 값으로 폼 pre-fill

## 4. 상태 관리 설계

```
URL 쿼리 파라미터
└── missionaryId: string | null  — 선택된 선교 ID (새로고침 시 유지)

클라이언트 상태 (useState)
└── — (별도 없음, URL 기반으로 충분)

서버 상태 (React Query)
├── missionaries: Missionary[]          — 선교 목록 (SELECT 드롭다운용)
└── missionaryRegions: MissionaryRegion[]  — 연계지 목록 (선택된 선교 기준)

모달 상태 (overlay-kit)
├── MissionaryRegionFormModal            — 등록/수정 (mode: 'create' | 'edit', 초기값 전달)
└── DeleteMissionaryRegionModal          — 삭제 확인
```

## 5. 인터랙션 플로우

### 등록 플로우
```
[+ 연계지 등록] 클릭
  → MissionaryRegionFormModal 열림 (mode: 'create', missionaryId 전달)
  → 폼 입력 → [저장] 클릭
  → API: POST /missionaries/:id/regions
  → 성공: 모달 닫힘 + 목록 갱신 + Toast "연계지가 등록되었습니다"
  → 실패: 에러 Toast
```

### 수정 플로우
```
행의 [수정] 클릭
  → MissionaryRegionFormModal 열림 (mode: 'edit', 현재 데이터 pre-fill)
  → 폼 수정 → [저장] 클릭
  → API: PATCH /missionaries/:id/regions/:regionId
  → 성공: 모달 닫힘 + 목록 갱신 + Toast "연계지 정보가 수정되었습니다"
  → 실패: 에러 Toast
```

### 삭제 플로우
```
행의 [삭제] 클릭
  → DeleteMissionaryRegionModal 열림 (연계지 이름 표시)
  → [삭제] 클릭
  → API: DELETE /missionaries/:id/regions/:regionId
  → 성공: 모달 닫힘 + 목록 갱신 + Toast "연계지가 삭제되었습니다"
  → 실패: 에러 Toast
```

### 이탈 방지 (Dirty Guard)
```
폼에 변경사항이 있는 상태에서 [×] 또는 [취소] 클릭
  → "저장하지 않은 변경사항이 있습니다. 나가시겠습니까?" 확인 모달
  → [나가기]: 모달 닫힘 (변경사항 폐기)
  → [계속 편집]: 폼으로 복귀
```

## 6. 접근성 (a11y)

- 모달: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` 제목 연결
- 필수 필드: `aria-required="true"`, 에러 시 `aria-describedby`로 에러 메시지 연결
- [삭제] 버튼: `aria-label="OO교회 삭제"` (테이블 행 컨텍스트 포함)
- [수정] 버튼: `aria-label="OO교회 수정"`
- 포커스 트랩: 모달 열림 시 내부로 포커스 제한, 닫힘 시 트리거 버튼으로 복귀
- 키보드: `Escape`로 모달 닫기

## 9. 개발 구현 가이드 (PATCH 임시 처리)

PATCH API가 아직 미구현 상태이므로 다음 방식으로 처리한다:

```
// 수정 버튼 렌더 (ADMIN 전용)
// PATCH API 배포 전: disabled={true}
// PATCH API 배포 후: disabled={false} 로만 변경
<Button
  variant="outline"
  size="sm"
  disabled={true}  // ← BE PATCH 배포 시 제거
  title="현재 사용 불가 (준비 중)"
  onClick={() => openEditModal(region)}
>
  수정
</Button>
```

- 수정 모달 폼 컴포넌트 (`MissionaryRegionForm`) 는 완성 상태로 구현
- `useUpdateMissionaryRegion` 훅도 완성 상태로 구현
- BE PATCH 배포 후 `disabled={true}` 한 줄만 제거하면 즉시 활성화
